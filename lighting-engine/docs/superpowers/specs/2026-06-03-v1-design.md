# nectar-studio v1 — Design Spec

**Status:** approved (sections 1–5)
**Date:** 2026-06-03
**Author:** captured from brainstorming session between founder and engine team

---

## 0. Goal

Ship a working v1 of nectar-studio that takes a designer's DWG (ceiling + furniture plans) and produces, **for one designer-selected room at a time**, a revised reflected ceiling plan + revised furniture/floor plan + lux uniformity numbers — delivered as a web-rendered SVG package the designer can review, screenshot, and share with their client.

v1 is *one room per project*. Multi-room ships in v1.1.

---

## 1. Repository layout & architecture

Single monorepo, moving the existing studio frontend from `nectar-viz` into `nectar-studio`:

```
nectar-studio/
├── lighting-engine/             # Python 3.11, uv, FastAPI service
│   └── src/lighting_engine/
│       ├── parser/              # ezdxf-based DWG/DXF parsing (exists, refining)
│       ├── digest/              # spatial digest (exists)
│       ├── lighting/            # lumen method + placement (exists)
│       ├── brief/               # NEW: LLM brief layer (Step 5)
│       ├── render/              # NEW: SVG renderers
│       ├── lux/                 # NEW: uniformity calculation
│       └── api/                 # NEW: FastAPI routes
├── studio/                      # Next.js frontend (moved from nectar-viz)
│   └── app/studio/...           # existing flow: upload → rooms → … → pack
└── shared/openapi.yaml          # generated contract between the two
```

**Runtime topology.** `studio` deploys to Vercel; `lighting-engine` deploys to Fly.io / Render (or similar) as a separate FastAPI service. The two communicate over HTTP. Studio uses an OpenAPI-generated TypeScript client. CORS is configured on the engine side.

**Why FastAPI:** native async, OpenAPI schema generation, pydantic v2 (already used everywhere in the engine).

## 2. Data flow & API surface

### 2.1 Map of studio screens to engine endpoints

```
SCREEN                       ENDPOINT (lighting-engine)                  PAYLOAD
─────────────────────────    ────────────────────────────────────────    ─────────────────────────────
/studio/upload          ─▶   POST  /api/projects                         multipart: ceiling.dwg + furniture.dwg
                                                                          ↓
                                                                          { project_id,
                                                                            rooms: [{id, name, type, dims,
                                                                              polygon, doors, windows,
                                                                              confidence, tier}] }

/studio/rooms           ─▶   GET   /api/projects/{pid}/rooms             { rooms: [...with status...] }

/studio/room-basics     ─▶   GET   /api/projects/{pid}/rooms/{rid}       full room IR + preview SVG
                             POST  /api/projects/{pid}/rooms/{rid}       { ceiling_height_m, ceiling_type,
                                                                            window_orientation, occupants,
                                                                            floor_finish, wall_finish }

/studio/walls           ─▶   GET   /api/projects/{pid}/rooms/{rid}/walls
                             POST  /api/projects/{pid}/rooms/{rid}/walls/{idx}
                                                                          { confirm: bool, doors: […],
                                                                            windows: […] }

/studio/furniture       ─▶   POST  /api/projects/{pid}/rooms/{rid}/furniture
                                                                          { mood, activities, … }

/studio/brief           ─▶   POST  /api/projects/{pid}/rooms/{rid}/brief
                                                                          { intent_mood, time_of_use, … }

/studio/generating      ─▶   POST  /api/projects/{pid}/rooms/{rid}/generate
                                                                          → { job_id }   (async)
                             GET   /api/jobs/{job_id}                     { status, result_url? }

/studio/pack            ─▶   GET   /api/projects/{pid}/rooms/{rid}/plan   { rcp_svg, furniture_svg,
                                                                            lux_uniformity, design_rationale,
                                                                            fixture_schedule, warnings }
```

### 2.2 Backend pipeline at `POST /generate`

```
   1. Merge user clarifications into the parsed Room IR
      (override dims; add ceiling height; set finishes; override
      windows/doors per wall).
                              ↓
   2. Compute spatial digest for the confirmed room.
                              ↓
   3. LLM brief (Step 5):
        digest + clarifications  ─▶  Claude Opus 4.7  ─▶  RoomBrief
                                                          (target lux, CCT,
                                                           zones, rationale)
                              ↓
   4. Deterministic placement (Step 4 exists; extends to task / accent /
      decorative layers):
        room polygon + brief  ─▶  fixture positions on ceiling (RCP) +
                                  suggested floor/table-lamp positions.
                              ↓
   5. Lux uniformity:
        0.5m grid over the room polygon → point-source lux per cell →
        mean / min / max / uniformity ratio.
                              ↓
   6. SVG renderers:
        - render/rcp.py:        original ceiling features + new lighting layer
        - render/furniture.py:  original furniture + lamp markings
                              ↓
   7. Assemble PlanResponse → store in SQLite → return job result.
```

### 2.3 State

SQLite at `lighting-engine/data/state.db` for v1 (per-project + per-room + per-job rows). Postgres if/when we ship multi-tenant.

### 2.4 Async generation

`POST /generate` enqueues a FastAPI BackgroundTask. LLM call dominates latency (~5–15s on Opus 4.7 with adaptive thinking + prompt cache hit). Studio polls `GET /api/jobs/{id}` from `/studio/generating`.

## 3. Room filter + user-clarification merge

### 3.1 Filter (which parsed rooms reach the picker)

| Tier | Room types | Treatment |
|---|---|---|
| **First-class** (full v1 design flow) | Living room, Drawing / formal living, Dining, Bedroom / Master bedroom, Family lounge, Bar, Kitchen, Study / WFH, Puja room | `/studio/room-basics` → `/studio/walls` → `/studio/furniture` → `/studio/brief` → full RCP + furniture plan + lux uniformity |
| **Generic recommendation** (deferred to a later phase) | Foyer / entrance, Lobby, Passage, Dress room | Returned in the picker with `tier: "generic"` but the generation pipeline for these is *deferred to v1.1*. v1 only generates plans for first-class rooms. |
| **Hidden in v1** | Toilet, bathroom, storage, utility, washing area, courtyard, terrace, staircase, balcony, shaft | Not shown in picker |

Classification happens server-side after parsing; `GET /api/projects/{pid}/rooms` only returns first-class + generic-tier rooms.

### 3.2 ConfirmedRoom IR — the merge target

Each POST from the studio updates a single `ConfirmedRoom` blob in SQLite keyed by `(project_id, room_id)`:

```python
ConfirmedRoom {
  # from parser (immutable after upload)
  id, name, type_inferred, polygon_inferred, doors_parsed, windows_parsed,
  furniture_parsed, ceiling_features_parsed

  # from /studio/room-basics
  type_confirmed: RoomType            # may override type_inferred
  length_m, width_m
  ceiling_height_m                    # required, no parser inference
  ceiling_type                        # false / flat / sloped / mixed
  main_window_orientation: Direction  # N / S / E / W
  occupants: list[Occupant]           # kids / young_adult / adult / elderly
  floor_finish, wall_finish           # light / mid / dark

  # from /studio/walls
  walls: [{index, doors_confirmed, windows_confirmed, notes}]

  # from /studio/furniture
  furniture_notes: str

  # from /studio/brief
  intent_mood: Mood                   # cozy / productive / wind_down / entertain
  activities: list[str]
  time_of_use: list[TimeOfDay]        # morning / evening / late_night

  # provenance — for debugging
  provenance: {field_name: "parser" | "user"}
}
```

### 3.3 Merge rules

1. **User > parser** for any field the user touched.
2. **Parser default** for fields the user hasn't touched. The studio pre-fills with parser inferences so the user is *editing*, not creating.
3. **Required-by-generate fields**: `ceiling_height_m`, `type_confirmed`, `main_window_orientation`. If any are missing at `POST /generate`, return `400` with the list of missing fields → studio routes the user back to the relevant screen.
4. **Provenance is tracked but not displayed in v1**. Logged for debugging ("why did the engine recommend cool CCT? oh, parser inferred kitchen and user didn't override").

## 4. LLM brief layer + output rendering

### 4.1 RoomBrief schema

The LLM emits semantic **zones**, never coordinates. The deterministic placement code translates each zone into fixture positions on the polygon.

```python
class LightingLayer(StrEnum):
    ambient = "ambient"
    task = "task"
    accent = "accent"
    decorative = "decorative"

class Zone(BaseModel):
    layer: LightingLayer
    purpose: str                 # "ambient over center" / "task above dining table"
    cct_k: int
    cri_min: int
    fixture_type: str            # "downlight" / "pendant" / "cove" / "floor_lamp"
    position_hint: str           # "center of ceiling" / "above dining table" /
                                 # "wall A near window"

class RoomBrief(BaseModel):
    # Machine fields → deterministic placement
    target_lux_ambient: float
    cct_main: int
    fixture_preference: Literal["warm-bias", "cool-bias", "mixed"]
    layers_needed: list[LightingLayer]
    zones: list[Zone]
    warnings: list[str]                  # "no daylight — increase ambient"

    # Text fields → report rendering
    design_rationale: str                # 1-3 paragraphs
    design_notes: list[str]
    floor_lamp_suggestions: list[Zone]   # for furniture plan
    table_lamp_suggestions: list[Zone]   # for furniture plan
```

### 4.2 Cached system prompt (~5–8K tokens)

```
[CACHED PREFIX — frozen across all rooms]
  • Lighting science: lumen method constants, IS 3646 targets per room type
  • Indian-residential preferences: CCT/lux per room type, layered framework
  • Cultural context: family lounges, puja rooms, bar accents, etc.
  • Layer principles: ambient / task / accent / decorative
  • Output format: RoomBrief JSON schema (pydantic-serialized)

[PER-ROOM USER MESSAGE — varies]
  • RoomDigest (walls, openings, adjacency, daylight side)
  • Designer brief (intent_mood, activities, occupants, time_of_use, finishes)
  • Structured numbers (target_lux from standards table; fixture catalog options)
  • Free-text designer notes
```

Frozen prefix sourced from `docs/research/lighting/01–10.md`. Cache verification per request via `usage.cache_read_input_tokens`. Silent-invalidator audit per `shared/prompt-caching.md` (no `datetime.now()` in prefix, deterministic JSON, etc.).

**Model:** Claude Opus 4.7 with `thinking: {type: "adaptive"}` and `output_config: {effort: "high"}`. Structured outputs enforced via `output_config.format` against the `RoomBrief` schema.

**RAG layer:** deferred to v1.1. v1 inlines the full lighting-research corpus into the cached system prompt — small enough to fit, no vector-DB infra cost.

### 4.3 Brief → placement integration

```python
# Today (Step 4): single ambient layer, room-type → standard.
standard = get_lux_standard(room.type)
fixtures = compute_ambient_layer(room, digest)

# v1 (Step 4 + Step 5):
brief = generate_room_brief(room, digest, confirmed_room)   # LLM call
fixtures = []
for zone in brief.zones:
    if zone.layer == LightingLayer.ambient:
        fixtures.extend(compute_ambient_layer(room, digest, brief=brief))
    elif zone.layer == LightingLayer.task:
        fixtures.extend(compute_task_layer(room, digest, zone, brief))
    elif zone.layer == LightingLayer.accent:
        fixtures.extend(compute_accent_layer(room, digest, zone, brief))
    elif zone.layer == LightingLayer.decorative:
        fixtures.extend(compute_decorative_layer(room, digest, zone, brief))
```

New code: `compute_task_layer`, `compute_accent_layer`, `compute_decorative_layer`. Each interprets a zone's `position_hint` into actual coordinates using the digest's furniture data.

### 4.4 SVG output

**Revised RCP SVG (`render/rcp.py`):** polygon outline + parsed ceiling features (beams, soffits as dashed outlines) + new lighting layer (downlights as filled circles, coves as dashed lines, pendants/chandeliers as ornate icons). Color-coded by CCT (warm = orange, cool = blue). Wall directions labeled (N/S/E/W). Header strip with fixture count + total wattage + main CCT.

**Revised furniture SVG (`render/furniture.py`):** polygon + parsed furniture shapes + suggested floor lamps (filled triangles) and table lamps (smaller filled triangles) with intent annotations ("reading", "ambient corner", etc.).

### 4.5 Lux uniformity (`lux/uniformity.py`)

Point-source model: `E = I × cos(θ) / d²`. 0.5m grid over the room polygon, work plane at 0.8m above floor. Sum contributions from all fixtures per cell.

```python
class LuxStats(BaseModel):
    mean_lux: float
    min_lux: float
    max_lux: float
    uniformity: float           # min / mean
    target_lux: float           # from brief.target_lux_ambient
    meets_target: bool          # mean_lux >= 0.9 * target_lux
```

**v1 returns summary numbers only.** No heatmap overlay (v1.1).

### 4.6 Fixture schedule (in v1)

Group fixtures by SKU + count + spec. Maps directly to the studio's `FixtureScheduleTable` component.

```python
class FixtureRow(BaseModel):
    sku: str
    name: str
    wattage_w: float
    lumens: float
    cct_k: int
    cri: int
    beam_angle_deg: float
    count: int
```

### 4.7 PlanResponse (what studio gets at `GET /plan`)

```python
class PlanResponse(BaseModel):
    project_id: str
    room_id: str
    rcp_svg: str                     # full <svg>...</svg> string
    furniture_svg: str               # ditto
    lux_uniformity: LuxStats
    fixture_schedule: list[FixtureRow]
    design_rationale: str            # from brief.design_rationale
    design_notes: list[str]
    warnings: list[str]
    metadata: dict                   # { generated_at, model_used,
                                     #   prompt_cache_hit_rate }
```
